# The Heights Handyman

> **Done right. Done clean. Done on time.** — A fast, SEO-optimized marketing
> website for a handyman business serving South Euclid, University Heights,
> Cleveland Heights, and Beachwood, Ohio, with a special focus on
> **point-of-sale (POS) inspection violation repair.**

Built as a lightweight static site generated by a small Node script (no
framework), with a serverless contact-form handler that emails the owner.

A division of North Coast Residential Solutions LLC.

---

## Quick start

```bash
npm install          # install dependencies (nodemailer, for the contact form)
npm run build        # generate the site into /dist
npm run serve        # preview at http://localhost:3000
# or:
npm run dev          # build + serve in one step
```

The deployable site is the generated **`/dist`** folder.

## ✏️ Placeholders to update before launch

Edit **`src/data/site.js`** — search for `PLACEHOLDER`:

- **`url`** — your real production domain (canonical tags, sitemap, schema)
- **Address / ZIP** and map coordinates (a real address improves local SEO)
- **Social links** (optional)

Phone `(216) 284-9433`, email `ezra@theheightshandyman.com`, and the four
service cities are already set.

**Logo:** A lightweight, image-free "HH" gold badge sits in the header (see
`brandMark()` in `src/templates/layout.js`). Drop a real logo into
`public/images/` and swap that block if desired.

**Photos:** The hero, POS banner, and services images use branded
`placehold.co` placeholders. Replace the `src` URLs with real, free-to-use
stock photos (e.g. Unsplash):

- Home hero (`src/templates/pages/home.js`) — handyman with a tool belt
- POS banner (`posBanner()` in `src/templates/components.js`) — home exterior / inspection
- Services hero (`src/templates/pages/services.js`) — handyman at work

Keep the descriptive, keyword-relevant `alt` text.

Run `npm run build` after any change.

## 📨 Making the contact form send email

The contact form POSTs to a serverless function in `/api/contact.js`, which
emails submissions to the owner. Set these environment variables (see
`.env.example`):

```
SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS   # your mail provider
MAIL_FROM   # optional; defaults to SMTP_USER
MAIL_TO     # inbox that receives requests (ezra@theheightshandyman.com)
```

Until SMTP is configured, the form degrades gracefully: the visitor is shown a
friendly message asking them to call/text directly (no errors, no data loss).

> Anti-spam: the form includes a hidden honeypot field and server-side
> validation.

## 🚀 Deploying

### Vercel (recommended — serverless form works out of the box)
1. Push this repo to GitHub.
2. Import it in Vercel. `vercel.json` already sets the build command
   (`npm run build`), output directory (`dist`), and clean URLs.
3. Add the SMTP environment variables in **Project → Settings → Environment Variables**.
4. Deploy. The `/api/contact` function is detected automatically.

### Any static host (Netlify, Cloudflare Pages, S3, etc.)
- Build command: `npm run build` · Publish directory: `dist`
- For the email form you'll need a serverless/function equivalent, or point
  the form `action` at a form service. The handler logic lives in `/api`.

## 🗂 Project structure

```
build.js                 # static site generator (run via npm run build)
serve.js                 # local preview server + API routing
vercel.json              # host config (build, clean URLs, caching, headers)
.env.example             # SMTP config template
api/
  contact.js             # POST /api/contact  → emails estimate requests
  _lib/mailer.js         # shared nodemailer + validation helpers
public/                  # static assets copied verbatim into /dist
  css/styles.css         # the full design system (navy / gold / cream)
  js/main.js             # nav, AJAX form, area filter
  images/                # favicon + social share image (SVG)
src/
  data/
    site.js              # ⭐ company info + PLACEHOLDERS
    services.js          # trade categories (drives the tiles)
    locations.js         # the four service cities (drive the landing pages)
  templates/
    layout.js            # HTML shell: SEO head, header, footer
    components.js        # reusable sections, icons, JSON-LD builders
    pages/               # one module per page type
dist/                    # generated output (the website) — git-ignored
```

## Adding or editing a service city

Add a row to `src/data/locations.js` (name, county, type, nearby, character,
homes) and rebuild — a complete, unique, SEO-optimized landing page is
generated automatically.
